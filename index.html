2. Arquitectura Central

Traer la Instrucción a la CPU:

  1. (PC) → MAR: El contenido del Contador de Programa se envía al Registro de Direcciones de Memoria.
  2. Memoria → MDR: La instrucción se trae desde la memoria al Registro de Datos de Memoria.
  3. (MDR) → IR: La instrucción se copia al Registro de Instrucción para ser decodificada.
  4. PC + 1 → PC: Se incrementa el PC para apuntar a la siguiente instrucción.

La Unidad de Control decodifica la instrucción. Si requiere operandos, se buscan en memoria. Finalmente, la CU envía las señales a la ALU para realizar la operación, y el resultado se guarda en un registro o en memoria.

3. Unidad de Control

Interpreta las instrucciones y genera las señales que gobiernan al resto de componentes.

Control Cableado (Hardwired)

Implementado como un circuito secuencial que genera las señales de control directamente a partir del código de operación. Es muy rápido, pero complejo de diseñar y modificar. Típico de arquitecturas RISC.

Principio de funcionamiento: Un decodificador recibe la instrucción (IR), un contador de pasos de reloj y las banderas de estado (Flags). Esta información alimenta una lógica combinacional que produce las micro-órdenes (señales de control) en la secuencia correcta.

Control Microprogramado

Utiliza una memoria de control (ROM) que almacena secuencias de microinstrucciones (microprogramas). Cada instrucción de máquina ejecuta un microprograma. Es más lento, pero mucho más flexible y fácil de diseñar y extender. Típico de arquitecturas CISC.

4. Segmentación (Pipeline)

Técnica que solapa la ejecución de múltiples instrucciones. Idealmente, una instrucción en un pipeline de N etapas demora en promedio 1/N del tiempo que tardaría sin pipeline. Los problemas a evitar son los riesgos (hazards):

Riesgos Estructurales

Conflicto de Recursos

Dos etapas del pipeline necesitan usar el mismo recurso (ej. memoria) al mismo tiempo.

Solución: Duplicar recursos o insertar paradas.

Riesgos de Datos

Dependencia de Datos

Una instrucción depende del resultado de una instrucción anterior que aún no ha terminado.

Solución: Adelantamiento (Forwarding) o inserción de paradas.

Riesgos de Control

Conflicto por Saltos

Una instrucción de salto modifica el PC e impide saber cuál es la siguiente instrucción a cargar.

Solución: Predicción de saltos, Salto Retardado o Registros de Ciclos (Loop Buffer).

6. RISC vs. CISC

Los procesadores RISC surgen para cerrar la "brecha semántica" entre los lenguajes de alto nivel y las arquitecturas. Se observó que la mayoría de programas usaban solo un pequeño subconjunto de instrucciones complejas, y que un conjunto de instrucciones reducido y optimizado podría ser más rápido al simplificar el hardware.

CISC (Complex)

  • Muchas instrucciones complejas (más de 100).
  • Control microprogramado.
  • Múltiples formatos de instrucción de longitud variable.
  • Gran cantidad de modos de direccionamiento.
  • Ejemplos: Intel x86, Motorola 68k.

RISC (Reduced)

  • Pocas instrucciones simples (la mayoría en 1 ciclo de reloj).
  • Control cableado (más rápido).
  • Formato de instrucción fijo y simple.
  • Solo LOAD/STORE acceden a memoria.
  • Ejemplos: ARM, MIPS, SPARC, PowerPC.

7. Jerarquía de Memoria

Organización de la memoria en niveles para optimizar velocidad, coste y capacidad.

Registros / Caché

Nivel más alto

Volátil. La más rápida, de menor capacidad y más cara. La caché almacena datos e instrucciones de uso frecuente para acelerar el acceso de la CPU.

Memoria Principal (RAM)

Nivel intermedio

Volátil. Velocidad, capacidad y coste intermedios. Almacena los programas y datos que se están utilizando activamente.

Memoria Secundaria

Nivel más bajo

No volátil. La más lenta, de mayor capacidad y más barata. Almacenamiento persistente (Discos Duros, SSD, Cintas).

8. Gestión de Entrada/Salida (E/S)

Técnicas para comunicar la CPU con los periféricos.

E/S Programada

Control por CPU

La CPU tiene control total. Espera constantemente (polling) a que el periférico esté listo. Ineficiente, ya que la CPU queda bloqueada.

E/S por Interrupciones

Aviso del Periférico

El periférico avisa a la CPU con una señal de interrupción cuando está listo. La CPU puede hacer otras tareas mientras tanto. Mucho más eficiente.

Acceso Directo a Memoria (DMA)

Transferencia Independiente

Un controlador DMA se encarga de transferir bloques de datos entre memoria y periféricos sin intervención de la CPU, liberándola por completo.

9. Paralelismo: Clasificación de Flynn

Se basa en el número de flujos de instrucciones y flujos de datos que se manipulan simultáneamente. Considera componentes como la Unidad de Control (CU), Unidad de Procesamiento (PU) y Módulos de Memoria (MM).

SISD

Single Instruction, Single Data

Una única unidad de control y una única unidad de procesamiento. Es la arquitectura Von Neumann tradicional (uniprocesador).

SIMD

Single Instruction, Multiple Data

Una instrucción se ejecuta simultáneamente sobre múltiples datos. Ideal para procesamiento vectorial y gráfico (ej: GPUs).

MIMD

Multiple Instruction, Multiple Data

Múltiples procesadores ejecutan diferentes instrucciones sobre diferentes datos. Se dividen según su organización de memoria en Memoria Compartida y Memoria Distribuida.

10. Interrupciones

Mecanismo que detiene temporalmente el programa actual para atender un evento prioritario.

Concepto y Proceso

Una señal que indica a la CPU que necesita atención. El proceso es: 1. Detener programa actual. 2. Guardar el estado de la máquina (PC, registros) en la pila. 3. Ejecutar la rutina de servicio de interrupción (ISR). 4. Restaurar el estado y reanudar el programa original.

Tipos de Interrupciones

Hardware: Generadas por dispositivos externos (ej. fin de E/S, teclado).
Software: Generadas por una instrucción (ej. llamadas al sistema).
Excepciones: Causadas por errores (ej. división por cero).

Gestión y Prioridad

Los sistemas tienen un sistema jerarquizado con niveles de prioridad. Una interrupción de mayor prioridad puede interrumpir a una de menor. Se usan máscaras de interrupción para habilitar/deshabilitar niveles (ej: en el 8085 con las instrucciones RIM y SIM).
